<p class="MsoNormal" style="text-align: center;" align="center">Linker Tutorial for
embedded systems</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">The purpose of this tutorial is to show how to read the
linker script file and also the GNU linker document.<span style="">  </span>It does not go into every aspect of the
linker (m68k-elf-ld) but provides enough information to understand the CodeSourcery
linker script file and to use it in embedded baremetal systems.</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">It is necessary to understand the linker script file so as
to ensure that only the code you want in the final executable is there and the
source of that final executable. Knowing what is in the final code ensures that
you have the smallest code necessary for your embedded system.<span style="">  </span>It also can provide insight as to how best to
use static ram and cache in your system.</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">First some background; when you compile your program into an
executable file, the compiler creates one or many position independent object
files where this PIC code is placed in output sections.<span style="">  </span></p>

<p class="MsoNormal"> </p>

<p class="MsoNormal" style="margin-left: 1in;">Position Independent Code (PIC)
code means that final address locations have not been resolved. Why you
ask?<span style="">  </span>On final linking which is what we
are addressing in this tutorial, you can choose to put your code at any
location within memory and so a function you wrote could be at location ox10 or
ox30.<span style="">  </span>You can decide where using the linker
script or let the linker do it automatically.</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">These output sections become the basis for the input
sections within the linker script. There is also additional code that is added
by way of the linker script file.<span style="">  </span>These
too have output sections.<span style="">  </span>This
additional code is used to configure the controller and then set up your
environment to execute your program from the main function.<span style="">  </span>This additional code comes from archive
object files such as libc.a, libcs3.a, libcs3coldfire.a, libcs3hosted.a,
etc.<span style="">  </span>I refer to these as archive files
because each file listed is a combination of several files.<span style="">  </span>To see what is in these files use the
following command.</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">C:\CodeSourcery\Sourcery G++
Lite\bin\m68k-elf-objdump.exe" -h -l -p -s -D<span style="">  </span>c:\CodeSourcery\Sourcery G++
Lite\m68k-elf\lib\libcs3.a" > c:\temp\objdump.txt</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">This file contains a number of object files.<span style="">  </span>Below are just a few:</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">start-sim.o</p>

<p class="MsoNormal">start_c.o</p>

<p class="MsoNormal">heap.o</p>

<p class="MsoNormal">demoem-reset.o</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">The linker is used to take your compiled source code and the
above archive files and put them in certain physical locations in the resulting
executable file and resolve function call addresses.<span style="">  </span>The linker does numerous other things but
these are it purpose.</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">There are a number of things to understand about the GNU linker
document. The first and most important thing to understand is the <span style=""> </span>“input sections” , “output sections” and the “SECTIONS”
phrases.<span style="">  </span>In particular the input
sections actually comes from the output sections of your source object code in
which libcs3.a is apart.<span style="">  </span>Put another
way, when you compile and link your program your compiler first creates one or
more object files sometimes denoted with a “.o” extension.<span style="">  </span>In these object files your code is broken
down into sections with names that are arbitrary but in most cases they are
.text. .data, .bss, etc.<span style="">  </span>Knowing the output
section names you can then use them to map to the output sections of your final
executable.<span style="">  </span>The period before the .text
is just part of the name and bears no other significance.</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">Now the resulting executable file is in some sort of
format.<span style="">  </span>It may be in ELF, S19, PE or
another format.<span style="">  </span>We will only be
considering ELF in our tutorial.<span style="">  </span>And so
the SECTIONS keyword defines the layout of the final executable.<span style="">  </span>Below shows the syntax for the SECTIONS
phrase.<span style="">  </span>Think of the contents of linker
file as being the skeleton layout similar to laying out a book.<span style="">  </span>Think of the SECTIONS as the body of the
document.</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal" style=""><span style="font-size: 9pt; font-family: "Courier New";">SECTIONS</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";">{</span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";">} /*End of sections*/</span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal" style="">There
is only one <span style="font-size: 9pt; font-family: "Courier New";">SECTIONS </span>phrase
in a linker file and within the curly brackets we then define the output
sections that go into the exactable. <span style="font-size: 9pt; font-family: "Courier New";"></span></p>

<p class="MsoNormal"> </p>

<p class="MsoNormal" style=""><span style="font-size: 9pt; font-family: "Courier New";">SECTIONS</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";">{</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">     </span>".text"
:</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">  </span><span style="">   </span>{</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">     </span>}</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">.data :</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style=""> </span><span style="">    </span>{</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">}</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">.bss.eh_frame :</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">  </span><span style="">   </span>{</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">}</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">.eh_frame_hdr :</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">{</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">}</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">.rodata :</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">{</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">}</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">.cs3.rambar :</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">{</span></p>

<p class="MsoNormal" style="text-indent: 0.5in;"><span style="font-family: "Courier New";">}</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";">} /*End of sections*/</span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal" style="">Within
the<span style="font-family: "Courier New";"> </span><span style="font-size: 9pt; font-family: "Courier New";">SECTIONS </span>phrase you
have<span style="font-size: 9pt; font-family: "Courier New";"> </span><span style="font-family: "Courier New";">".text", .data, .bss.eh_frame,
.eh_frame_hdr, .rodata, .rodata.<span style="">  </span></span>These
are just some<span style="font-family: "Courier New";"> </span>“output sections”
that are being defined for the resulting executable you are creating.<span style="">  </span>You will notice that <span style="font-family: "Courier New";">".text" </span>has quotes and the others do not, I
added the quotes to demonstrate that it is just a name and the period before it
has no special meaning.<span style="">  </span>The other output
sections could also be quoted.<span style="">  </span>What is
unique is that these names are arbitrary and you could call them “dog” and
“cat”.<span style="">  </span>The reason why you will see so
many examples with <span style="font-family: "Courier New";">.text and .data </span>is
that it is a holdover from the a.out binary format.</p>

<p class="MsoNormal" style=""> </p>

<p class="MsoNormal" style="">Now
each output section can have one or more input sections from the source object
files you have compiled and the added archive files.<span style="">  </span>Remember when you compiled your main.c file,
the linker also added some archive files at link time to get your application
to work on your microcontroller.<span style="">  </span>Think
of the input sections as the paragraphs to each chapter where the chapter is
the output sections e.g. <span style="font-family: "Courier New";">".text"
</span>and<span style="font-family: "Courier New";"> .data</span>.<span style="">  </span>You take paragraphs of information from your
research which in this case is your source object files and the added archive files.<span style="">  </span>The “<span style="font-family: "Courier New";">*(.text)</span>”
below is an input section that states “if any of my source object files and
archive files has a .text section place the code within this section
here.”<span style="">  </span>The asterisk (*) means any.</p>

<p class="MsoNormal" style=""> </p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";">".text" :</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">  </span><span style="">   </span>{</span></p>

<p class="MsoNormal" style="margin-left: 0.5in; text-indent: 0.5in;"><span style="font-family: "Courier New";">*(.text)
/*line 1*/</span></p>

<p class="MsoNormal" style="margin-left: 0.5in; text-indent: 0.5in;"><span style="font-family: "Courier New";">*(.text1
.rdata) /*line 2*/</span></p>

<p class="MsoNormal" style="margin-left: 0.5in; text-indent: 0.5in;"><span style="font-family: "Courier New";">*(.text2)
*(.rdata) /*line 3*/</span></p>

<p class="MsoNormal" style="margin-left: 0.5in; text-indent: 0.5in;"><span style="font-family: "Courier New";">Foo.o (.text3)
/*line 4*/</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">     </span>}</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal">Line 2 from above states to put .text1 and .rdata input
section code here randomly.<span style="">  </span>Line 3
states put .text2 and then .rdata input section code here.<span style="">  </span>Line 3 could have put the .rdata input
section on the next line down. And finally line 4 states put .text3 input
section code from the Foo.o file here.<span style=""> 
</span>Also because each line is sequential, code will be added
sequentially.<span style="">  </span></p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">Let’s turn to where we want to put our code in memory.<span style="">  </span>We have several methods of placing our <span style="font-family: "Courier New";">".text" </span>output section<span style="font-family: "Courier New";"> </span>and the other output sections<span style="font-family: "Courier New";"> </span>in memory but I will only demonstrate
one.<span style="">  </span>First we will define our memory
regions in our microcontroller.<span style="">  </span>This is
done with the below phrase.</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal"><span style="font-family: "Courier New";">MEMORY</span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";">{</span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";"><span style="">  </span>rom (rx) : ORIGIN = 0x00000000, LENGTH = 2M</span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";"><span style="">  </span>ram (rwx) : ORIGIN = 0x40000000, LENGTH = 16M</span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";"><span style="">  </span>rambar (rwx) : ORIGIN = 0x80000000, LENGTH =
16K</span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";">}</span></p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">We have given each memory range a name: rom, ram and
rambar.<span style="">  </span>So now we will tell the linker
to put <span style="font-family: "Courier New";">".text" </span>in a
particular location with the<span style="font-family: "Courier New";"> “>rom</span>”
phrase.</p>

<p class="MsoNormal" style=""><span style="font-size: 9pt; font-family: CMTT9;"> </span></p>

<p class="MsoNormal" style=""><span style="font-size: 9pt; font-family: CMTT9;"> </span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";">".text" :</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">  </span><span style="">   </span>{</span></p>

<p class="MsoNormal" style="margin-left: 0.5in; text-indent: 0.5in;"><span style="font-family: "Courier New";">/*input
sections here*/</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">     </span>}
>rom</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal" style="">This
will place the<span style="font-family: "Courier New";"> ".text" code
in rom starting at 0x00000000</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal">Note that there is only one MEMORY phrase and it comes
before the SECTIONS phrase.</p>

<p class="MsoNormal" style=""> </p>

<p class="MsoNormal" style="">I
have also seen this syntax<span style="">  </span>“>rambar
AT>rom” in the codesourcy linker files but am unaware of how this works.
RESEARCH NEEDED.</p>

<p class="MsoNormal" style=""> </p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";">".text" :</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">  </span><span style="">   </span>{</span></p>

<p class="MsoNormal" style="margin-left: 0.5in; text-indent: 0.5in;"><span style="font-family: "Courier New";">/*input
sections here*/</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"><span style="">     </span>} </span>>rambar
AT>rom</p>

<p class="MsoNormal" style=""> </p>

<p class="MsoNormal" style="">Above
I have shown that I can place my input sections sequentially in my output
section however each input section may not start at the most optimum location
so I can adjust them individually by doing any of the following:</p>

<p class="MsoNormal" style=""> </p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";">. = ALIGN (8);</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";">_bdata = (. + 3) & ~ 3;</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";">variable = ALIGN(0x8000);</span></p>

<p class="MsoNormal" style=""><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal">I suggest only using the first statement since it adjusts
the location counter to a location where the next input section listed after it
will start.<span style="">  </span></p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">We now turn to the “.” which is called the location
counter.<span style="">  </span>Think of this as a program
counter whose address increments as input sections are added sequentially to
each output section.<span style="">  </span>You may see the
location counter manipulated as above to change its address or get its address.<span style="">  </span>The ld.pdf demonstrates why you would want
its address.</p>

<p class="MsoNormal"> </p>

<p class="MsoNormal">Note. Do not confuse this dot “.” with those added to the

<p class="MsoNormal"><span style="font-size: 13pt; font-family: CMBX12;">Summary.</span></p>

<p class="MsoNormal"><span style="font-size: 13pt; font-family: CMBX12;"> </span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";">When you create your final executable program it goes through several
steps.<span style="">  </span>First your source code is
compiled into position independent object code that has output sections.<span style="">  </span>Next, these output sections become the basis
for input sections of your linker script.<span style=""> 
</span>Your linker script then places these various output sections now input
sections into output sections for your final executable.<span style="">  </span>Since we are dealing with the elf format and
using similar section names then talking about output and input sections
sometimes become ambiguous. </span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";"> </span></p>

<p class="MsoNormal"><span style="font-family: "Courier New";">The linker uses dot “.” location pointer to identify and resolve
addresses within the final executable. What was once PIC code in now position
dependent but know because it was defined by the linker script. We can also
manipulate the dot location pointer to move to various locations to place our
code.</span></p>
